2) Схема БД (SQL для создания таблиц)
Файл со схемой: db/schema.sql

PRAGMA foreign_keys = ON;

-- Таблица-справочник 1 (главный): departments (Отделы)
CREATE TABLE departments (
  id            INTEGER PRIMARY KEY,          -- id задаётся вручную (не полагаемся на авто-генерацию)
  name          TEXT    NOT NULL,             -- допускаются одинаковые name при разных id
  description   TEXT    NOT NULL DEFAULT '',  -- многострочный текст
  founded_date  TEXT    NOT NULL,             -- дата: YYYY-MM-DD
  headcount     INTEGER NOT NULL CHECK (headcount >= 0),
  annual_budget NUMERIC(12,2) NOT NULL CHECK (annual_budget >= 0)
);

-- Таблица-справочник 2 (зависимый): projects (Проекты)
CREATE TABLE projects (
  id               INTEGER PRIMARY KEY,          -- id задаётся вручную
  department_id    INTEGER NOT NULL,             -- FK -> departments.id
  title            TEXT    NOT NULL,
  summary          TEXT    NOT NULL DEFAULT '',
  start_date       TEXT    NOT NULL,             -- дата: YYYY-MM-DD
  planned_end_date TEXT    NOT NULL,             -- дата: YYYY-MM-DD
  priority         INTEGER NOT NULL CHECK (priority BETWEEN 1 AND 5),
  project_budget   NUMERIC(12,2) NOT NULL CHECK (project_budget >= 0),
  CONSTRAINT fk_projects_departments
    FOREIGN KEY (department_id)
    REFERENCES departments(id)
    ON UPDATE RESTRICT
    ON DELETE RESTRICT
);

Пояснение по внешним ключам и удалению:
- Связь: projects.department_id -> departments.id
- ON DELETE RESTRICT означает: нельзя удалить отдел, если на него ссылаются проекты. При этом проекты из зависимого справочника не удаляются автоматически (нет каскадного удаления).
